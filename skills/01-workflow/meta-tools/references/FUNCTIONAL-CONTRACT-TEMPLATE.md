## Common Integration Failures

These are the most frequently missed handler connections:

### All Platforms

| Element Pattern     | Common Issue                | Fix                          |
| ------------------- | --------------------------- | ---------------------------- |
| Icon-only buttons   | Missing handler entirely    | Add click/press handler      |
| Dropdown options    | onChange not connected      | Wire to state/URL update     |
| Modal triggers      | Modal component missing     | Create modal + setState      |
| Action menus        | Menu items without handlers | Connect to service functions |
| Form submissions    | No onSubmit or action       | Add form action/handler      |
| Pagination controls | Buttons with no effect      | Wire to page state           |

### Expo/React Native Specific

| Element Pattern          | Common Issue                | Fix                                       |
| ------------------------ | --------------------------- | ----------------------------------------- |
| Using `onClick`          | Wrong handler for RN        | Change to `onPress`                       |
| `<button>` elements      | Not valid in RN             | Use `<Pressable>` or `<TouchableOpacity>` |
| Missing `<Text>` wrapper | Text outside Text component | Wrap strings in `<Text>`                  |
| Web-style forms          | `<form>` doesn't exist      | Use state + submit handler                |

### Telegram Mini App Specific

| Element Pattern        | Common Issue               | Fix                                         |
| ---------------------- | -------------------------- | ------------------------------------------- |
| No haptic feedback     | Buttons feel unresponsive  | Add `hapticFeedback.impactOccurred()`       |
| Using HTML buttons     | Not Telegram-native        | Use `@telegram-apps/telegram-ui` components |
| Missing MainButton     | Primary action not visible | Wire to `mainButton.onClick()`              |
| BackButton not handled | User can't navigate back   | Add `backButton.onClick()` handler          |

### Discord Activity Specific

| Element Pattern         | Common Issue       | Fix                           |
| ----------------------- | ------------------ | ----------------------------- |
| SDK not initialized     | Commands fail      | Check `useDiscordSdk()` hook  |
| Missing activity update | Activity not shown | Call `commands.setActivity()` |
| No participant sync     | State out of sync  | Use `subscribe()` for events  |
