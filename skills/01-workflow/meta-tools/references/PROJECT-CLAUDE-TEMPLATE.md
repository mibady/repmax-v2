## Common Tasks

### Add a new feature

1. Check if hooks/actions already exist in `{HOOKS_PATH}` and `{ACTIONS_PATH}`
2. If database changes needed, create migration first
3. Build component using existing patterns from similar pages
4. Wire up to existing hooks/actions
5. Run `/test` to verify
6. Commit with descriptive message

### Fix a bug

1. Identify the affected component/page
2. Check the hook/action it uses
3. Trace data flow: Component → Hook → API/Action → Database
4. Fix at the appropriate layer
5. Run `/test` to verify
6. Commit with descriptive message

### Add database migration

1. Create migration in `{MIGRATIONS_PATH}`
2. Run `/db migrate` to apply
3. Run `/db types` to regenerate TypeScript types
4. Update affected hooks/actions if needed
